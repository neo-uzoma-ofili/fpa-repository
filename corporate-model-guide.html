What This Guide Covers

This document captures the architecture, data sources, logic, key dependencies, and known quirks of the Neo Corporate Financial Model. Its purpose is to reduce single points of failure and accelerate onboarding for new team members.

The corporate model is the single source of truth for all consolidated financial forecasting at Neo Financial. It produces the company's income statement, balance sheet, and cash flow statement across four scenarios, and drives all program-level variance analysis and board reporting.

2
Model Version History

Forecast cycles follow the naming convention FXX'YY — e.g. F03'26 = forecast number 3 for 2026. A new model file is created each time a cycle is locked. Variants within a cycle (Base, Target, Board) are always separate files and are never overwritten.

📁
All model files are stored in the FP&A Folder in Google Drive. The Forecast Model Directory lists every cycle with its scenario and direct file link.

Known Issues & Quirks
⚠️
Forecast label inconsistency The Cover tab, output tab (col B), and fcst_output tab may show different cycle labels (e.g. F02'26 on Cover, F04-26 in output, F03-26 in fcst_output). Individual tabs are often pre-stamped ahead of the official cycle lock. Always confirm the authoritative version via Cover + Updates tab.
🚫
Consolidated TB — never sort The tab carries a hard warning. Sorting breaks all downstream GL formula lookups across the entire model.
🔴
fcst_output #REF! errors The "Neo" brand filter rows have broken references. Use only "All" brand rows for any downstream automation or data pulls.
🔴
Receivables [All] active accounts row (2024) Shows "413,421,557" — clearly a formula error. Do not use this figure directly.
⚠️
Covenants + Cash Flow Fcst date reference broken The current-date reference shows #N/A. Cash to Dry Land figures may not reflect today's date — verify against the CFS tab directly.
⚠️
Two exchange rate assumptions Investor tabs use CAD/USD = 1.35; Board Deck tab uses 1.425. Always confirm which rate applies before converting figures for presentations.
💡
Temp tabs are scratch space Tabs prefixed "Temp:" are working scratch tabs and are not part of the locked model. Contents may be stale or mid-edit.

Onboarding Checklist

For a new FP&A analyst — complete these in order, ideally with a senior analyst present for steps 5 and 10.

  • Obtain read access to the Corporate Financial Models folder in Google Drive
  • Review the model version index to understand the forecast cycle cadence (F01–F12 per year)
  • Open the most recent locked model; start with Cover → Updates → Control Panel
  • Read the Drivers tab to understand cohort mechanics — start with the Unsecured section
  • Trace one metric end-to-end: e.g. Interchange Revenue from Growth Intake → Drivers → P&L → output tab
  • Review the Consolidated TB tab to understand how GL actuals enter the model (do not sort)
  • Read the Macro Fcst tab to understand interest rate assumptions
  • Review the Forecast Notes Rolling Context Log for active forecast adjustments and known persistent variances
